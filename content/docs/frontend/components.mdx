# Components

The INBO Frontend uses a well-organized component structure with reusable, type-safe React components.

## Component Organization

### Directory Structure

```
components/
├── auth/              # Authentication components
├── layout/            # Layout components
├── inbox/             # Inbox/Email components
├── discover/          # Discovery components
├── profile/           # Profile components
├── analytics/         # Analytics components
├── reading/           # Reading components
├── seo/               # SEO components
└── [shared]/          # Shared components
```

## Component Patterns

### Server vs Client Components

**Server Components** (default):
- No `'use client'` directive
- Rendered on server
- No interactivity
- Can import server-only modules

**Client Components**:
- `'use client'` directive
- Rendered on client
- Interactive features
- Can use hooks, event handlers

### Component Composition

```typescript
// Compose components
<InboxCard
  email={email}
  onFavorite={handleFavorite}
  onRead={handleRead}
/>
```

### Props Interface

```typescript
interface InboxCardProps {
  email: Email;
  onFavorite: (id: string) => void;
  onRead: (id: string, isRead: boolean) => void;
}
```
